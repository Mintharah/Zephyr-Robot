All peripheral threads block on their semaphore and push results to the 16-deep result_q. Mailbox pattern: one pending command per thread.

ASCII command protocol
CMD:DIO_SET:<pin>:<val>
Drive a GPIO pin high or low. Pin 0–3 on GPIOB PB3–PB6.
CMD:DIO_SET:0:1 → OK:DIO_SET
CMD:DIO_GET:<pin>
Read back the logical level of a GPIO pin.
CMD:DIO_GET:1 → OK:DIO_GET:1:1
CMD:ADC_READ
Single-shot 12-bit ADC read on PA0. Returns raw counts (0–4095).
CMD:ADC_READ → OK:ADC_READ:2048
CMD:PWM_SET:<period_ns>:<pulse_ns>
Set TIM3 CH3 (PB0) period and pulse width in nanoseconds.
CMD:PWM_SET:1000000:500000 → OK:PWM_SET
CMD:UART_SEND:<text>
Send text out USART2 (PA2 TX → PA3 RX jumper). Clears RX buffer first.
CMD:UART_SEND:HELLO → OK:UART_SEND
CMD:UART_RECV
Receive one line looped back on USART2 RX. 300 ms timeout.
CMD:UART_RECV → OK:UART_RECV:HELLO
CMD:SPI_XFER:<hex bytes>
Full-duplex SPI transfer on SPI1. Hex bytes space-separated; returns RX as compact hex.
CMD:SPI_XFER:DE AD BE EF → OK:SPI_XFER:DEADBEEF

Errors return ERR:<reason>. All frames are newline-terminated. Queue depth = 16 commands; depth = 16 results. Unknown commands respond ERR:UNKNOWN.

Dynamic architecture — command execution flow
RX path — host to peripheral
1USART1 interrupt fires on each received byte. cmd_rx_thread buffers into rx_line[128] until \n.
2cmd_parse() tokenises the line into a cmd_t struct (type + up to 5 arguments). On success, k_msgq_put() enqueues it to cmd_q (non-blocking).
3cmd_dispatch_thread blocks on k_msgq_get(). It switches on cmd.type, copies the command into the target thread's mailbox slot (e.g., dio_cmd), then calls k_sem_give() on the matching semaphore.
4The woken peripheral thread (prio 4) executes the hardware operation — GPIO toggle, ADC conversion, PWM reconfigure, SPI transceive, or UART bridge.
TX path — result back to host
5The peripheral thread formats a char[64] response (OK:… or ERR:…) and calls k_msgq_put(&result_q, …).
6result_tx_thread (prio 3) unblocks from k_msgq_get(&result_q) and transmits each character via uart_poll_out() on USART1.
7The Python TestbenchLib reads one line over serial.readline() (2 s timeout), checks the OK: prefix, strips the payload, and returns it to the Robot Framework keyword.
Concurrency model
→Priority-3 threads (RX, dispatch, TX) run at higher priority than priority-4 peripheral threads — command routing never starves behind a slow peripheral operation.
→Mailbox pattern (one pending cmd_t slot per thread) keeps memory bounded. Dispatch overwrites the slot then gives the semaphore atomically; no race window exists because peripheral threads are lower priority and cannot preempt.
→CONFIG_TIMESLICING=y ensures same-priority threads (all four prio-4 peripherals) share CPU time if multiple semaphores are signalled in quick succession.
Robot Framework test suites
U test_uart.robot
  • UART loopback returns sent text
  • UART empty response timeout
SPI test_spi.robot
  • SPI single byte transfer
  • SPI multi-byte transfer
  • SPI zero byte
ADC test_adc.robot
  • ADC in: within supply range (0–3300 mV)
  • ADC in: readings stable (Δ < 200 mV)
  • ADC out: half scale (1650 mV) reads back
  • ADC out: low level (500 mV) reads back
  • ADC out: monotonic voltage sweep
PWM test_pwm.robot
  • PWM out: 50% / 25% / full / off
  • PWM in: captures period (≈1 ms)
  • PWM in: captures 50% duty
  • PWM in: tracks 25/50/75% sweep
DIO test_dio.robot
  • Pair 0–1 set high, readback
  • Pair 0–1 set low, readback
  • Pair 2–3 set high, readback
  • Pair 2–3 set low, readback
  • Both pairs toggle (FOR loop)

ADC tests average 8 samples to suppress PWM ripple. SPI tests use a PA7→PA6 (MOSI→MISO) jumper so the master reads back its own bytes. DIO tests rely on PB3↔PB4 and PB5↔PB6 GPIO loopback wired on the board.
